# Reviewer Workflow v4

## R0: Initialization (NEW WINDOW REQUIRED)
- Do NOT read Developer chat history
- Use ONLY code + documents

## R1: Scope Check
Determine:
- Is change within module scope?

## R2: Design Alignment
Check:
- PRD
- RUBRIC
- ARCHITECTURE
- SCHEMA

## R3: Code Quality
Check:
- Logic correctness
- Edge cases
- Error handling
- Naming conventions

## R4: Data Consistency
Check:
- Schema compliance
- Data risks

## R5: Progress Alignment (ENHANCED)
Check:
- code ≈ PROJECT_PROGRESS.md
- code ≈ SNAPSHOT.md

## R6: Risk Analysis
Identify:
- Regression risks
- Fragile logic

## R7: Output (MANDATORY)
Follow PROTOCOL.md

## R8: Strict Review Rule
Reviewer MUST maintain a critical stance.
Any deviation from PRD or SCHEMA MUST be marked as BLOCKING_ISSUE even if code runs.
